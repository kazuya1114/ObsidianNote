
# ロジスティック回帰

******ある事象が起こる確率を予測するための機械学習手法**で、主に**分類問題**に使う。

名前に「回帰」とあるが、目的は

> **データをクラス（例：0か1）に分類すること**

![ロジスティック回帰](../pic/11.png)
![ロジスティック回帰2](../pic/12.png)

## 計算式

$$y = B₀ × B₁^{x₁} \times B₂^{x₂}$$

$$ý = \frac{1}{1 + e^{-z}}$$$$z = b₀ + b₁x₁ + b₂x₂$$

## 分類分け例

b₀ + b₁x₁ + b₂x₂ > 0：生活習慣病
b₀ + b₁x₁ + b₂x₂ <= 0：健康

## フィーチャースケーリング（特徴量の標準化）実行の判断基準

標準化の理由は、メッシュを揃えて正しい分析をするため（yの値は標準化してもしなくてもよい（標準化されたxの値をもとに算出されるため））。
最も簡単な判断基準は

> **距離・勾配を使うアルゴリズム → スケーリング必要**
> **木ベースアルゴリズム → 不要**

| アルゴリズム         | フィーチャースケーリングの要否 | 理由     |
| -------------- | --------------- | ------ |
| ロジスティック回帰      | 要               | 勾配降下法  |
| SVM / SVR      | 要               | 距離計算   |
| KNN            | 要               | 距離計算   |
| ニューラルネット       | 要               | 勾配最適化  |
| PCA            | 要               | 分散ベース  |
| 線形回帰（L1/L2正則化） | 要               | 係数の影響  |
| 決定木            | 否               | 条件分岐のみ |
| 回帰木            | 否               | 同上     |
| ランダムフォレスト      | 否               | 木ベース   |
| XGBoost        | 否               | 木ベース   |
| LightGBM       | 否               | 木ベース   |

1. **データの分布**:
    - データが正規分布に従っている場合は、標準化（平均0、標準偏差1）を行うことが推奨されます。
2. **アルゴリズムの特性**:
    - サポートベクターマシン（SVM）やK平均法など、距離に基づくアルゴリズムでは、フィーチャースケーリングが必要になることが多いです。
    - 一方で、重回帰分析などは必ずしもフィーチャースケーリングが必要ではありませんが、精度向上に寄与する場合があります。
3. **目的に応じた判断**:
    - 各アルゴリズムの背景やデータセットの特性に基づいてフィーチャースケーリングを行うかどうかを吟味することが重要です。
    - 特定の目的に応じてスケーリングが必要であるかどうかを検討し、必要に応じて試行するのが良いでしょう。1. **データの分布**:
    - データが正規分布に従っている場合は、標準化（平均0、標準偏差1）を行うことが推奨されます。


## ロジスティック回帰の実装

```python
# ロジスティック回帰分析
import numpy as np
import matplotlib.pyplot as plt
import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LinearRegression
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import OneHotEncoder
from sklearn.preprocessing import PolynomialFeatures
from sklearn.preprocessing import StandardScaler
from sklearn.svm import SVR
from sklearn.tree import DecisionTreeRegressor
from sklearn.ensemble import RandomForestRegressor
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import r2_score
from sklearn.metrics import confusion_matrix, accuracy_score
from matplotlib.colors import ListedColormap

# 分析前の前処理
dataset = pd.read_csv('data/Social_Network_Ads.csv')
X = dataset.iloc[:, :-1].values  # 独立変数（Age列とEstimatedSalary列）
Y = dataset.iloc[:, -1].values   # 従属変数（Purchased列)

# 訓練用データとテスト用データセットへの分割
# 　test_size=0.2：データ全体の20%をテスト用データに使用（残りの80%を訓練用データに使用）
# 　random_state=0：乱数シードを0に設定（毎回同じ分割結果になるようにするための設定）
X_train, X_test, y_train, y_test = train_test_split(X, Y, test_size = 0.25, random_state = 0)

# フィーチャースケーリング（特徴量の標準化）
sc = StandardScaler()
X_train = sc.fit_transform(X_train) # Xの標準化
X_test = sc.transform(X_test) # テスト用データの標準化（訓練用データの平均と標準偏差を使用してテスト用データを標準化）

# モデルの学習
classifier = LogisticRegression()
classifier.fit(X_train, y_train) # 学習（回帰曲線の決定（＝切片と傾きの決定））

# 結果の予測
classifier.predict(sc.transform([[30, 87000]])) # 年齢が30歳で年収が87,000ドルの人が商品を購入するかどうかを予測（予測値は標準化されているため、逆変換して元のスケールに戻す）

# 結果の比較
y_pred = classifier.predict(X_test) # テスト用データの予測値を算出
print(np.concatenate((y_pred.reshape(len(y_pred), 1), y_test.reshape(len(y_test), 1)), axis=1)) # 予測値と実際の値を並べて表示

# 混同行列の作成
cm = confusion_matrix(y_test, y_pred) # 混同行列の作成
print(cm) # 混同行列の表示
print(accuracy_score(y_test, y_pred)) # 正解率の算出

# 訓練用データセットの可視化
X_set, Y_set = sc.inverse_transform(X_train), y_train # 訓練用データを元のスケールに逆変換

# グリッドの作成
X1, X2 = np.meshgridc(
    np.arange(start = X_set[:, 0].min() - 10, stop = X_set[:, 0].max() + 10, step = 0.25)
    , np.arange(start = X_set[:, 1].min() - 1000, stop = X_set[:, 1].max() + 1000, step = 0.25)
)

  

# 決定境界の描画
plt.contourf(
    X1
    , X2
    , classifier.predict(sc.transform(np.array([X1.ravel(), X2.ravel()]).T)).reshape(X1.shape)
    , alpha = 0.75
    , cmap = ListedColormap(('red', 'green'))
)

# グラフの範囲を設定
plt.xlim(X1.min(), X1.max())
plt.ylim(X2.min(), X2.max())

# 訓練用データの散布図を描画
for i, j in enumerate(np.unique(Y_set)):
    plt.scatter(X_set[Y_set == j, 0], X_set[Y_set == j, 1], c = ListedColormap(('red', 'green'))(i), label = j)

# グラフのタイトルと軸ラベルを設定
plt.title('Logistic Regression (Training set)')
plt.xlabel('Age')
plt.ylabel('Estimated Salary')
plt.legend()
plt.show()

# テスト用データセットの可視化
X_set, Y_set = sc.inverse_transform(X_test), y_test # テスト用データを元のスケールに逆変換

# グリッドの作成
X1, X2 = np.meshgridc(
    np.arange(start = X_set[:, 0].min() - 10, stop = X_set[:, 0].max() + 10, step = 0.25)
    , np.arange(start = X_set[:, 1].min() - 1000, stop = X_set[:, 1].max() + 1000, step = 0.25)
)

# 決定境界の描画
plt.contourf(
    X1
    , X2
    , classifier.predict(sc.transform(np.array([X1.ravel(), X2.ravel()]).T)).reshape(X1.shape)
    , alpha = 0.75
    , cmap = ListedColormap(('red', 'green'))
)

# グラフの範囲を設定
plt.xlim(X1.min(), X1.max())
plt.ylim(X2.min(), X2.max())

# テスト用データの散布図を描画
for i, j in enumerate(np.unique(Y_set)):
    plt.scatter(X_set[Y_set == j, 0], X_set[Y_set == j, 1], c = ListedColormap(('red', 'green'))(i), label = j)

# グラフのタイトルと軸ラベルを設定
plt.title('Logistic Regression (Test set)')
plt.xlabel('Age')
plt.ylabel('Estimated Salary')
plt.legend()
plt.show()
```

## ロジスティック回帰の回帰曲線が直線になる理由

![ロジスティック回帰3](../pic/13.png)

ロジスティック回帰分析の結果、回帰曲線が直線となる理由は、ロジスティック回帰のモデルが線形部分とシグモイド関数を用いているから。

1. **線形モデル**: ロジスティック回帰では、まず説明変数の線形結合を計算します。この線形結合は「独立変数×係数 + 切片」という形で表されます。
2. **シグモイド関数**: 得られた線形モデルの結果に対してシグモイド関数を適用します。このシグモイド関数の形状は、出力を0から1の範囲に収束させ、二値分類の確率を表現します。しかし、元の線形モデルが直線の形であり、出力の0または1に対応する境界を決定します。
3. **境界の解釈**: 結果として直線が生じ、特定の閾値で出力が異なるクラスに分類されるため、この境界がある種の直線として表現されます。つまり、シグモイド関数で圧縮された出力の確率を使って、クラスを決定するための直線を引くことになります。


# K近傍法

**新しいデータに最も近いデータ（近傍）をK個探し、その情報をもとに予測する機械学習アルゴリズム**。
簡単に言うと

> **「似ているデータは似た結果になる」という考え方で予測する方法**

![K近傍法](../pic/14.png)
## 手順

1. 対象とするデータの周辺から選択する数（K）を選ぶ
2. ユークリッド距離を使い、Kがどのクラスターに一番近いか決める
3. Kの中で最も多いカテゴリーの数を選択する
4. 3で選択したクラスターを対象とするデータが属するクラスターとする

![K近傍法2](../pic/16.png)

## ユークリッド距離

![ユークリッド距離](../pic/15.png)
### 式

$$\sqrt{(X₂ - X₁)^2 + (y₂ - y₁)^2}$$

## K近傍法の実装

```python
# K近傍法
import numpy as np
import matplotlib.pyplot as plt
import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LinearRegression
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import OneHotEncoder
from sklearn.preprocessing import PolynomialFeatures
from sklearn.preprocessing import StandardScaler
from sklearn.svm import SVR
from sklearn.tree import DecisionTreeRegressor
from sklearn.ensemble import RandomForestRegressor
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import r2_score
from sklearn.metrics import confusion_matrix, accuracy_score
from matplotlib.colors import ListedColormap
from sklearn.neighbors import KNeighborsClassifier

# 分析前の前処理
dataset = pd.read_csv('data/Social_Network_Ads.csv')
X = dataset.iloc[:, :-1].values  # 独立変数（Age列とEstimatedSalary列）

# 　test_size=0.25：データ全体の25%をテスト用データに使用（残りの75%を訓練用データに使用）
# 　random_state=0：乱数シードを0に設定（毎回同じ分割結果になるようにするための設定）

X_train, X_test, y_train, y_test = train_test_split(X, Y, test_size = 0.25, random_state = 0)

# フィーチャースケーリング（特徴量の標準化）

X_train = sc.fit_transform(X_train) # Xの標準化
X_test = sc.transform(X_test) # テスト用データの標準化（訓練用データの平均と標準偏差を使用してテスト用データを標準化）

# モデルの学習
# 　K近傍法のインスタンスを作成（n_neighborsは近傍点の数（Kの数）、metricは距離の計算方法、pは距離の計算に使用するパラメータ（2 = 2次元においての距離を計算））

# 結果の予測
classifier.predict(sc.transform([[30, 87000]])) # 年齢が30歳で年収が87,000ドルの人が商品を購入するかどうかを予測（予測値は標準化されているため、transformで逆変換して元のスケールに戻す）

# 結果の比較

# 訓練用データセットの可視化
X_set, Y_set = sc.inverse_transform(X_train), y_train # 訓練用データを元のスケールに逆変換

# グリッドの作成
X1, X2 = np.meshgridc(
    np.arange(start = X_set[:, 0].min() - 10, stop = X_set[:, 0].max() + 10, step = 0.25)
    , np.arange(start = X_set[:, 1].min() - 1000, stop = X_set[:, 1].max() + 1000, step = 0.25)
)

# 決定境界の描画
plt.contourf(
    X1
    , X2
    , classifier.predict(sc.transform(np.array([X1.ravel(), X2.ravel()]).T)).reshape(X1.shape)
    , alpha = 0.75
    , cmap = ListedColormap(('red', 'green'))
)

# グラフの範囲を設定

plt.xlabel('Age')
plt.ylabel('Estimated Salary')
plt.legend()
plt.show()
```
