# DevOpsでよく利用されるツール

- コンテナ化技術
  - Docker：ユーザーフレンドリーなコンテナ化ツール
  - Kubernetes：複数コンテナを自動管理・調節するツール（コンテナオーケストレーション）
- CI/CD
  - GitHub Actions：GitHubでCI/CDを自動化するツール
  - ArgoCD：Kubernetesで動くGitベース運用のツール

# DevOpsとは

## 定義

ソフトウェア開発手法の一つ。
開発（Development）と運用（Operations）を組み合わせたかんばん語であり、開発担当者と運用担当者が連携して協力する（さらに両担当者の境目もあいまいにする）開発手法をさす。
ソフトウェアを迅速にビルドおよびテストする文化と環境により、確実なリリースを以前よりも迅速に高い頻度で可能とする組織体制の構築を目指している。

## バリューストリーム

製品やサービスを顧客に届けるために必要な一連の活動のこと。

## DevOpsの４つの柱（CAMS）

- 文化（culture）
- 自動化（automation）
- メトリクス（metrics）
- 共有（sharing）

## DevOpsの４つの指標（Four Keys）

- デプロイの頻度
- 変更のリードタイム
- 変更時の障害発生
- サービス復元時間
